Alternate version of the C++ Standard Template Library (STL)

In the product installation subfolder /inc/cpp/dlib5, there is an alternate version of STL that was used in earlier versions of IAR Embedded Workbench. This version of STL is simpler and can under favorable circumstances produce smaller code than the current version. In particular, this goes for the vector, map, multimap, set, and multiset implementations, which are better at sharing code between different instantiations than the current version. The old non-templated versions of string and complex are also included. Note that there is no support for I/O, and that you cannot instantiate any of the collection classes on a type from the std namespace.

You use this version of STL by specifying dlib5/ in your #include directive (for example: #include <dlib5/vector>). All names in the dlib5 header files are in the dlib5 namespace, so you need to either use using namespace dlib5; or, specify the dlib5 prefix for each feature name. Example: dlib5::vector<int>

The following headers are included:

#include <dlib5/complex>
#include <dlib5/deque>
#include <dlib5/hash_map>
#include <dlib5/hash_set>
#include <dlib5/list>
#include <dlib5/map>
#include <dlib5/queue>
#include <dlib5/set>
#include <dlib5/slist>
#include <dlib5/stack>
#include <dlib5/string>
#include <dlib5/vector>
